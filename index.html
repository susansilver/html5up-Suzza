A Creator Who Knows Their Way Around the Back-End

The ways we use Content Management Systems (CMS) are changing as write once publish everywhere is a reality for all.

I've been using WordPress as my CMS for most of my career as a freelance creator. I was heavily influenced by the book "Content Everywhere: Strategy and Structure For Future-Ready Content" by Sara Wachter-Boettcher. This book was published in 2014 near the time that responsive design was becoming the standard for websites. Wachter-Boettcher was inspired by this, to develop a system to orchestrate content that allowed writers to easily publish and have the engineers make it available everywhere. Typically, through the use of a content API.

JAMstack has made this easy to use for individuals. It is short for a tech stack that includes JavaScrpt, API, and Markdown. There are now tools out there like Eleventy that make templating a site easy. It ships HTML and a minimal amount of CSS and JavaScript to the browser.

This allowed me to take full advantage as a self-taught dev to convert my WordPress site to a static site generator called Astro. What sets this one apart is that it ships no JavaScript to the browser unless you tell it to. It uses island architecture and partial hydration. The client directives allow components to become interactive at the time you specify. It is really neat and it makes my mathematics blog load super fast.

This technology has really inspired me to become a back-end developer. I want to be able to build my own content API from scratch. I also want to learn how to work with databases so that I can build apps that have persistant data. That is what I will be working on this year. I see it as fullfiling my needs as a creator to have full control of my content. You can follow me on this journey by reading my blog at Suzza.dev.
