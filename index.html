A Creator Who Knows Content from Front to Back

Write once publish everywhere is a reality for all.

One book that had a huge influence on me is "Content Everywhere" by Sara Wachter-Boettcher. I read that book in 2014! Almost a decade ago. That is when I first heard of an API, application programming interface. This is a way that we can grab blogs, images, and other data from a CMS and import it anywhere that we want! This seemed out of reach to me because you needed a back-end dev to build a content API. So they were used by enterprise business and startups. Not so much for individuals or small businesses. We had to use WordPress.

This has changed in the past few years as the JAMstack and static site generators (SSG) have made it much easier to use API for building a blog. JAMstack refers to a tech stack made of JavaScript, API, and Markup. Websites are templated and assets can be served via a Content Delievery Network (CDN). Content is pre-buiilt and loaded in advance. This allows websites to load quickly all around the world! Popular static site generators include Jekyll, Eleventy, and Hugo.

This year, I converted my WordPress blog to an Astro website. Astro is an SSG with some unique ideas. It uses island architecture and partial hydration. This means that components become interactive when you want them to. For example, only load scripts if a component is visible. If a user never browses to see that component then nothing runs. This does not interfere with any other compontent on the page. Hence each one is its own island.

I am a completely self-taught developer. I managed to build my math blog into an Astro site. This puts a SSG into the hands of solopreneurs and small business. The technology has never been easier to use! You can still continue to use WordPress. That is what is meant by Headless WordPress. It decouples the back-end from the front-end. You use WordPress as a pure CMS using the REST API to grab your content. You can then build your front-end using any framework you choose, even Astro will work.

I have discovered a new love of back-end development. I want to learn how to build a content API and create apps that use a database to keep presistent data. You can follow my journey at Suzza.dev.
